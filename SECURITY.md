# Security policy

Do not disclose a suspected vulnerability in a public issue, discussion, pull
request, or commit.

Use **Security → Advisories → Report a vulnerability** in this repository.
GitHub private vulnerability reporting sends the report only to repository
administrators and security managers while it is being triaged.

Until the first production release, only the current `main` branch is supported.
Reports should include the affected path or component, impact, reproduction
steps, and a minimal proof that does not contain real credentials or customer
data.
