# ADR 0023: Canonical PNU는 표준 대지구분 체계(1/2)로 통일한다

- Status: Accepted
- Date: 2026-07-08
- Evidence boundary: 시점별 행 수, 실행 ID, 체크섬, 조인 결과는
  [루트 ADR-0007](../../../../docs/adr/0007-public-code-private-operations-boundary.md)의
  비공개 운영 증거 경계를 따른다.

## Context

PNU 11번째 자리인 대지구분은 원천에 따라 의미가 다르다.

- **표준 지적 PNU**: `1` = 일반, `2` = 산
- **허브 건축물대장 원본**: `0` = 대지, `1` = 산, `2` = 블록

V-World, 경매, 필지 경계, 마커 앵커, 벡터 타일, by-PNU API는 표준 지적 PNU를 사용한다.
반면 건축물대장 원본의 대지구분을 그대로 이어 붙이면 허브 전용 표현이 된다. 이 두 표현을
같은 `pnu` 컬럼에 허용하면 소비자마다 변환 로직을 만들고, 블록 코드를 일반 필지로 잘못
축약할 수 있다.

이전 구현들은 같은 문제를 서로 다른 폴백으로 처리했다. 따라서 변환은 소비자 로직이 아니라
Foundation Platform 경계의 단일 계약이어야 한다.

## Decision

1. **Canonical Silver, serving, 공개 계약의 `pnu`는 표준 지적 체계만 담는다.**
   허브 원본 변환은 shared-kernel의 단일 SSOT 함수에서만 수행한다.
   - 허브 `0` → 표준 `1`
   - 허브 `1` → 표준 `2`
2. **허브 `2`인 블록은 대응하는 표준 PNU가 없으므로 `pnu = NULL`이다.**
   원본 대지구분과 블록 여부는 lineage로 보존하고 일반 또는 산 필지로 뭉개지 않는다.
3. **내부 대장 조인 키와 공개 PNU를 분리한다.**
   `register_parcel_key`는 허브 원본으로 조립한 내부 키이며 항상 존재할 수 있지만 PNU가
   아님을 계약에 명시한다. 전유부와 표제부 같은 동일 원천 내부 조인은 이 키를 사용하고,
   외부 필지 조인만 표준 `pnu`를 사용한다.
4. **재유입을 자동 차단한다.**
   - Silver 품질 게이트는 `pnu IS NULL OR substring(pnu, 11, 1) IN ('1', '2')`를 강제한다.
   - shared-kernel `Pnu` 파서는 길이뿐 아니라 대지구분 도메인을 검증한다.
   - `pnu IS NULL`인 대장 행은 원본 대지구분이 블록인 경우와 일치해야 한다.
5. **공개 핀 계약에 표현 체계를 명시한다.**
   Gongzzang 핀 계약과 Foundation 계약은 표준 지적 체계를 명시하고, 블록은 by-PNU 조회가
   불가능하다는 점을 설명한다.

## Migration and cutover

변환 함수, 전유부·표제부 조립, `register_parcel_key` 계약, writer, Spark 키, 소비자 핀을 하나의
호환 변경 단위로 다룬다. 내부 조인의 한쪽만 바꾸는 배포는 허용하지 않는다.

Iceberg 스냅샷을 유지한 채 새 Silver 스냅샷을 생성하고, 다음 불변식을 모두 확인한 후에만
active pointer를 승격한다.

1. 총 행 수가 이전 스냅샷과 같다.
2. 새 표준 `1`, 표준 `2`, `NULL` 파티션이 각각 이전 허브 `0`, 허브 `1`, 허브 `2` 파티션과
   정확히 대응한다.
3. 분류 상태와 링크 방법의 분포가 변하지 않는다.
4. `register_parcel_key` 내부 조인이 이전 내부 조인 결과를 보존한다.
5. 외부 매처는 소비자별 변환 셔밍 없이 표준 PNU로 조인한다.
6. 실제 소비자 계약 테스트가 표준 PNU 조회와 블록의 조회 불가 의미를 확인한다.

게이트 실패 시 이전 Iceberg 스냅샷과 이전 active pointer로 복귀한다. Bronze 원본과 Bronze
라인 기반 식별자는 변경하지 않으므로 승인·override 참조는 보존된다.

## Verification evidence boundary

이 ADR은 의미와 승격 불변식의 SSOT이며, 특정 실행의 완료 기록은 아니다. 실행 ID, 전후 행 수,
파티션별 수치, 조인 결과, 체크섬, 소요 시간, 실제 소비자 호출 결과는 비공개 운영 증거에
기록한다. 현재 구현 또는 컷오버 완료 여부는 코드, 마이그레이션, 최신 검증 실행을 함께
확인해야 하며 과거 실행 결과만으로 추론하지 않는다.

## Consequences

- (+) 표준 소비자는 변환 없이 Foundation PNU를 사용할 수 있다.
- (+) 표준 PNU가 없는 블록이 별도 해결이 필요한 클래스로 명확히 드러난다.
- (+) 변환 SSOT와 도메인 가드가 허브 표현의 재유입을 기계적으로 막는다.
- (-) Silver 계약에 `register_parcel_key`가 추가되며 두 키의 의미를 계속 설명해야 한다.
- (-) 전환 시 파생 체크섬과 소비자 픽스처를 새 계약으로 다시 생성해야 한다.

## Non-goals

- immutable Bronze 원본 재작성
- 블록의 주소 기반 표준 지번 해석
- PNU를 사용하지 않는 별도 층별개요 조인 방식 변경
