# V-World 수집 기능 계약

이 문서는 Foundation Platform이 사용하는 V-World 논리 데이터셋의 **채널 기능**을
정의한다. 공급자의 전체 카탈로그를 복제하거나 특정 실행 시점의 상태를 기록하는
문서가 아니다.

- 수집 채널 선택의 근거와 재검증 조건은
  [ADR 0018](../../adr/0018-vworld-collection-channel-strategy.md)이 소유한다.
- 데이터셋 selector, Bronze source slug, 식별자와 허용 수집 lane의 기계 판독 정본은
  [`public-source-endpoint-catalog.v1.json`](../public-source-endpoint-catalog.v1.json)이다.
- 실제 조회 범위, 응답 건수, 공급자 날짜, 파일 식별자, 화면·API 표본과 실행 결과는
  [루트 ADR 0007](../../../../../docs/adr/0007-public-code-private-operations-boundary.md)에
  따라 비공개 운영 증거에 보관한다.

## 채널 의미

| 채널 | 반환값 | Foundation Platform 용도 |
| --- | --- | --- |
| 벌크 파일 | 전체 파일과 공급자가 지원하는 변경 파일(SHP/CSV) | 전국 Bronze 수집의 기본값 |
| 속성 API | 속성 JSON/XML | 서빙, 통계, 제한된 비교, 승인된 예외 |
| 공간 API | 도형과 속성(GeoJSON/GML) | 서빙, 제한된 비교, 승인된 예외 |
| WMS·지도 타일 | 렌더링된 이미지 | 표시 전용; Bronze 수집 금지 |

`변경 파일`은 공급자가 독립된 변경 산출물을 제공한다는 기능 계약이다. UI에 필터가
있다는 사실만으로 지원한다고 추론하지 않는다. 지원하지 않는 데이터셋은 공급자 갱신
표식을 감시하고 새 전체 파일을 수집한 뒤, 캐노니컬 키와 콘텐츠 해시로 하류 변경을
계산한다.

## 변경 절차

1. 공식 공급자 문서와 제한된 probe로 채널의 반환 형식과 기능을 확인한다.
2. 요청·응답 표본과 실행 결과는 비공개 운영 증거에 저장한다.
3. Foundation Catalog 소유자가 기능 변경과 ADR 0018의 재검증 조건을 검토한다.
4. selector나 수집 lane이 바뀌면 기계 정본인
   `public-source-endpoint-catalog.v1.json`과 그 계약 테스트를 함께 변경한다.
5. 이 표에는 승인된 기능 결론만 반영한다. 날짜, 건수, 파일명, 요청 ID와 실행 상태는
   기록하지 않는다.

## 운영 불변식

- Bronze 원본은 불변이며 CreateOnly다.
- 데이터셋마다 acquisition watermark는 하나다.
- 공급자 변경 파일이 검증되지 않은 경우 전체 파일을 변경 파일처럼 취급하지 않는다.
- API 키와 계정별 endpoint는 런타임 설정으로만 주입한다.
- 지적 벌크 도형과 API 도형은 서로 다른 SRID일 수 있으므로 수신 메타데이터와 변환
  단계에서 SRID를 명시한다.
- 라이선스와 재배포 조건은 수집·변환·공개 전에 데이터셋별로 확인한다.
