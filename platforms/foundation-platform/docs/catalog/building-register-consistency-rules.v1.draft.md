# 건축물대장 일관성 검사 규칙 카탈로그 v1 (DRAFT — 소유자 승인 전)

- **상태**: 초안. 소유자 승인 + silver 스키마 바인딩 전까지 실행 금지
- **작성**: 2026-07-03 (gongzzang 세션에서 초안, foundation-platform 소유로 이관 예정)
- **방법론 근거**: Wikidata 제약 분류를 건축물대장 필드에 번역하고 Osmose 형식처럼
  위반 반환 SQL + 심각도 + 수정 제안으로 표현한다. 이 문서와 실행 SQL/테스트가 공개 계약이다.
- **실행 계약**: 규칙 1개 = "위반 행을 반환하는 SQL 1개". 반환 0행 = 통과. 전수 실행(샘플링 금지 — Google TFX 선례). severity: `error`(확실한 모순) / `warn`(판정 필요) / `info`(참고 플래그)
- **테이블/컬럼명은 placeholder** — silver 스키마 확정 시 바인딩. `silver.br_title`(표제부), `silver.br_floor`(층별개요), `silver.br_unit`(전유부), `silver.br_sub_parcel`(부속지번), `silver.land_char`(토지특성), `silver.land_right`(대지권등록), `silver.zone`(용도지역지구)

## 판정 전 0순위 (모든 규칙 공통)

위반 발견 시 판정 규칙 적용 **전에** 반드시: **자기 수집 누락 대조** — bronze 원본(ledger/manifest)에 해당 행이 있는데 silver에서 빠졌는지 확인. 범인 1순위는 우리 파이프라인이다. (Google KBT의 "추출 오류 vs 원천 오류 분리"의 실무 번역)

## 예외 처리 (Wikidata exception-to-constraint 패턴)

- 규칙마다 `exceptions` 목록(entity key 배열)을 데이터로 관리 — 규칙을 끄지 않고 예외를 등재
- 규칙 비활성화는 `deprecated` 플래그로 (삭제 금지 — 이력 보존)
- 오탐 신고는 위반 레코드에 `false_positive_reported` 필드로 축적 → 규칙 개선 루프

## 1장. 대장 내부 (건축물대장 자체 정합)

| ID | 유형(Wikidata) | sev | 검사 | 판정 힌트 |
|---|---|---|---|---|
| BR-C01 | conflicts-with | error | 표제부.지상층수 < MAX(층별.지상 층번호) — 층별에 표제부보다 높은 층 존재 | 0순위→명세>요약→옥탑 산입(下記)→전유부 교차→증축 시차 |
| BR-C02 | conflicts-with | warn | 표제부.지상층수 > MAX(층별.지상 층번호) — 표제부가 더 높음 (소유자 예시: 21 vs 20) | 층별 누락 가능성 우선(0순위 강화), 옥탑 미기재 |
| BR-C03 | conflicts-with | error | 지하층수 ↔ 층별 지하층 동일 검사 (BR-C01/02의 지하판) | 동일 |
| BR-C04 | distinct-values | error | 같은 건물 내 (층구분, 층번호) 중복 — "지하 1층이 두 개" | 명세 자체 오류, 병합/증축 이력 확인 |
| BR-C05 | item-requires-statement | warn | 전유부 존재 → 층별개요 존재 | 수집 순서/누락 |
| BR-C06 | value-requires-statement | error | 전유부.층이 층별개요 층 집합에 없음 (21층 호실인데 층별엔 20층까지) | 전유부는 실존 강력 증거 — 층별 누락 우선 |
| BR-C07 | range | error | 층별 면적 ≤ 0, 층수 범위 밖(1~130), 음수 면적 | 원천 오류 플래그 |
| BR-C08 | one-of | error | 층구분코드가 허용 집합 밖 | 사전(정규화 라인)으로 라우팅 |
| BR-C09 | format | error | PNU 19자리·대장 PK 형식 위반 | 수집/파싱 오류 우선 |
| BR-C10 | difference-within-range | warn | 사용승인일 < 착공일, 또는 착공일 < 허가일 | 원천 입력 오류 |
| BR-C11 | 집계 대조 | warn | Σ(층별 면적) vs 표제부.연면적 — 허용오차 밖 | 옥탑·부속건축물 산입 차이가 합법 원인일 수 있음 → 예외 규칙 |
| BR-C12 | item-requires-statement | info | 총괄표제부 존재 ↔ 표제부 복수 동 정합 (집합건물) | 동 누락 |

**BR-C01/02/11 공통 법규 예외**: 건축법 시행령 제119조 — 옥탑 등 건축면적의 1/8 이하인 부분은 층수 산입 제외. 판정 규칙에 산입 계산을 먼저 넣지 않으면 가짜 경보 다발.

## 2장. 교차 대장 (건축물대장 ↔ 토지 계열, PNU 조인)

**전제**: 건물↔필지는 N:M (부속지번 경유). 대지면적 비교는 관련지번 합산 기준.

| ID | 유형 | sev | 검사 | 판정 힌트 |
|---|---|---|---|---|
| XR-C01 | value-requires-statement | error | 표제부 대지위치 PNU가 토지특성/지적에 부재 | 지번 변경 이력(토지이동) 대조 |
| XR-C02 | 계산 검증 | error | \|기재 건폐율 − 건축면적÷대지면적\| > ε | 세 필드 상호 검증 — 어느 필드가 이상인지 잔차로 판별 |
| XR-C03 | 계산 검증 | error | \|기재 용적률 − 용적률산정연면적÷대지면적\| > ε | 동일. 지하층 연면적 제외 규칙 주의 |
| XR-C04 | 이중 기재 대조 | warn | 표제부.대지면적 ↔ Σ토지특성.면적(관련지번) 불일치 | 관련지번 누락 우선 확인 |
| XR-C05 | conflicts-with | info | 지목 ∈ {전, 답, 임야, …} ∧ 건축물대장 존재 | 합법 예외 다수(농막 등) — 플래그만, 자동 판정 금지 |
| XR-C06 | conflicts-with | info | 용도지역(예: 전용공업) ↔ 표제부 주용도 법적 비정합 | 자문 플래그 — 법령 매핑 테이블 필요 |
| XR-C07 | 집계 대조 | warn | 집합건물 대지권 비율 합 ≉ 1 (대지권등록정보 ↔ 전유부) | 등록 누락/이전 중 |
| XR-C08 | contemporary | info | 실거래(수집 개시 후)의 층 표기가 층별개요에 부재 | 교차 증거 — 층별 누락의 외부 신호 |

## 운영 원칙 (요약)

1. 탐지는 전수·결정적. AI는 이 카탈로그의 어떤 규칙에도 등장하지 않는다 — AI는 판정 잔여물과 **새 규칙 초안**에만
2. 위반은 유형별 그룹핑 + 영향 건수 순 정렬 (Wikidata 리포트 + 사람 클릭당 가치 최대화)
3. 판정 결과는 canonical을 덮지 않고 확정값(golden value)+근거 사슬+플래그로
4. 자동 적용은 평가셋 정밀도 실측 후 임계값 위 구간만 (Amazon AutoKnow 90% 정밀도 운영점 선례)
5. 새 위반이 유형으로 뭉치지 않으면 규칙 추가 중단 — 잔여물은 AI+사람 경로
